# 🚨 立即创建数据库集合

## 问题原因

500 错误很可能是因为**数据库集合还没有创建**！

## ✅ 立即执行步骤

### 1. 在 CloudBase 控制台找到文档型数据库

**重要**：不要使用 MySQL，要使用**文档型数据库**！

在左侧导航栏中找到：
- **"数据库"** → **"数据集合"** 或
- **"文档型数据库"** 或
- **"Database"** → **"Collections"**

### 2. 创建以下集合（至少先创建这3个）

**立即创建（必须）**：
1. ✅ `bookings` - 预约信息
2. ✅ `feedbacks` - 反馈信息  
3. ✅ `users` - 用户信息

**可以稍后创建**：
4. `admins` - 管理员信息
5. `halls` - 展区信息
6. `collections` - 收藏信息
7. `certificates` - 证书信息
8. `visit_settings` - 时段设置
9. `ar_checkins` - AR打卡

### 3. 创建方法

对每个集合：
1. 点击 **"新建集合"** 或 **"+ 添加集合"** 按钮
2. 输入集合名称（例如：`bookings`）
3. 点击 **"确定"**
4. 重复以上步骤创建其他集合

### 4. 验证

创建完成后，应该能在列表中看到这些集合。

### 5. 测试

集合创建后，重新测试小程序功能：
- 创建预约 → 应该能成功
- 查看预约列表 → 应该能看到数据
- 提交反馈 → 应该能成功

## 💡 提示

- 集合名称必须**完全匹配**（区分大小写）
- 集合会在首次写入时自动创建，但提前创建可以避免权限问题
- 创建集合后，**不需要**重新部署服务器
