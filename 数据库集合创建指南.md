# 🔧 数据库集合创建指南

## ⚠️ 重要说明

**你当前打开的是 MySQL 数据库，但我们的代码使用的是文档型数据库（NoSQL）！**

这是两个不同的数据库系统：
- **MySQL**：关系型数据库，使用表和行
- **文档型数据库**：NoSQL数据库，使用集合和文档

## ✅ 正确的操作步骤

### 步骤 1：找到文档型数据库

在 CloudBase 控制台左侧导航栏中，找到并点击：
- **"文档型数据库"** 或 **"数据集合"** 或 **"Database"**

**不是** "MySQL 数据库"！

### 步骤 2：创建必要的集合

在文档型数据库控制台中，创建以下集合：

1. ✅ **bookings** - 预约信息
2. ✅ **feedbacks** - 用户反馈信息
3. ✅ **users** - 用户账号信息
4. ✅ **admins** - 管理员账号信息
5. ✅ **halls** - 展区信息
6. ✅ **collections** - 用户收藏信息
7. ✅ **certificates** - 电子证书信息
8. ✅ **visit_settings** - 参观时段设置
9. ✅ **ar_checkins** - AR打卡记录

### 步骤 3：创建集合的方法

1. 在文档型数据库控制台中，点击 **"新建集合"** 或 **"+ 添加集合"**
2. 输入集合名称（例如：`bookings`）
3. 点击 **"确定"** 或 **"创建"**
4. 重复步骤 1-3，创建所有必要的集合

### 步骤 4：验证集合创建

创建完成后，你应该能在集合列表中看到所有 9 个集合。

## 🎯 最小必需集合（先创建这些）

如果时间紧急，至少先创建以下 3 个集合：

1. ✅ **bookings** - 预约信息（最重要！）
2. ✅ **feedbacks** - 反馈信息
3. ✅ **users** - 用户信息

## 📝 集合创建后

集合创建完成后：
1. 代码会自动使用这些集合
2. 首次写入数据时，集合会自动配置
3. 不需要手动创建表结构（文档型数据库不需要）

## ❓ 如果找不到文档型数据库入口

如果左侧导航栏没有"文档型数据库"，请：
1. 检查环境是否正确（`prison-museum-dev-8e6hujc6eb768b`）
2. 尝试点击"数据库"或"Database"查看是否有子菜单
3. 或者在控制台顶部搜索"文档型数据库"或"数据集合"
