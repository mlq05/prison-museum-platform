# 📝 数据库集合创建步骤（详细说明）

## ⚠️ 重要提示

**你当前查看的是 MySQL 数据库，但代码使用的是文档型数据库（NoSQL）！**

需要切换到**文档型数据库**来创建集合。

## 🔍 如何找到文档型数据库

### 方法 1：左侧导航栏

在 CloudBase 控制台左侧导航栏中，找到：

```
📊 数据库
  └─ 📁 数据集合  ← 点击这个！
```

或者：

```
📊 Database
  └─ 📁 Collections  ← 点击这个！
```

### 方法 2：搜索功能

在控制台顶部搜索框输入：**"数据集合"** 或 **"Collections"**

## ✅ 创建集合步骤

### 1. 进入数据集合页面

点击左侧导航栏的 **"数据集合"** 或 **"Collections"**

### 2. 创建第一个集合：bookings

1. 点击页面上的 **"新建集合"** 或 **"+ 添加集合"** 按钮
2. 在弹出的对话框中：
   - 集合名称：输入 `bookings`
   - 点击 **"确定"** 或 **"创建"**
3. 创建成功后，`bookings` 会出现在集合列表中

### 3. 创建第二个集合：feedbacks

重复步骤 2：
- 集合名称：`feedbacks`
- 点击创建

### 4. 创建第三个集合：users

重复步骤 2：
- 集合名称：`users`
- 点击创建

### 5. （可选）创建其他集合

按需创建：
- `admins`
- `halls`
- `collections`
- `certificates`
- `visit_settings`
- `ar_checkins`

## 🎯 最小必需集合

**至少需要创建这3个**：
1. ✅ `bookings` - 预约信息
2. ✅ `feedbacks` - 反馈信息
3. ✅ `users` - 用户信息

## ✅ 验证创建成功

创建完成后，你应该能看到集合列表中有这些集合：
- bookings
- feedbacks
- users
- ...（其他你创建的集合）

## 🔄 创建后测试

集合创建完成后：
1. **不需要重新部署服务器**
2. 直接在小程序中测试：
   - 创建预约 → 应该能成功
   - 查看预约列表 → 应该能显示数据
   - 提交反馈 → 应该能成功

## ❓ 如果找不到"数据集合"

如果左侧导航栏没有"数据集合"，请：
1. 确认环境是否正确（`prison-museum-dev-8e6hujc6eb768b`）
2. 检查是否有"数据库"菜单，点击展开看子菜单
3. 尝试在控制台搜索"文档型数据库"

## 💡 提示

- 集合名称必须**完全匹配**（区分大小写）：`bookings`，不是 `Bookings` 或 `BOOKINGS`
- 文档型数据库**不需要**创建表结构，集合本身就是一个空的容器
- 数据会在首次写入时自动存储
